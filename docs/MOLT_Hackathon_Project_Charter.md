## 3. 立项目的

MOLT 旨在回应 AI 时代年轻人在求职中遭遇的信心危机。产品不提供标准化求职建议，而是用 AI 找到"三个月前和你一样焦虑、现在已经拿到 Offer"的人，把 TA 带到你面前。

> 不推荐岗位，推荐人。不给答案，给一个走过这条路的真人。

Hackathon 阶段目标是验证以下核心命题：

- 用户是否愿意通过三次卡片点击（零文字输入）完成画像构建
- AI 是否能找到"曾经和你相似、已经走出来的人"并令人信服
- 信号连接机制是否足以形成强记忆点而不变成低质量社交

## 4. 被忽略的问题

与超过50名高校学生的访谈中，我们发现求职焦虑的核心往往不是效率问题，而是**信心问题**：

> "像我这样学翻译的，转AI还来得及吗？"
> "投了47份简历没有回音，不是简历不好，是不知道这条路走不走得通。"
> "我不需要AI给我一份模板，我需要一个真实的人告诉我——我走过，你也行。"

没有任何产品在做：**基于求职节点的共鸣，把一个正在迷茫的人和一个走过同样路的人连在一起。**

## 5. 目标用户

- 被 AI 冲击、开始怀疑专业价值的在校生和应届生（Path A：挣扎）
- 正在寻找迁移方向、需要同类反馈与现实验证的探索者（Path B：挣脱）
- 已完成转型、愿意以公开轨迹帮助后来者的前辈型用户（Path C：回望）

## 6. 四大 AI 创新

### 创新一：跨AI记忆迁移

传统产品要了解用户，要么让用户填表，要么从零开始对话。MOLT 从用户已有的 AI 助手（ChatGPT/Claude/豆包）中搬运记忆——用户粘贴一段 AI 助手对自己的200字描述，MOLT Agent 解析为结构化画像，绕过冷启动。据我们所知，目前没有产品将此作为核心交互环节。

### 创新二：语义聚类匹配（异时态）

MOLT 的匹配不是"找和你一样的人"，而是"找曾经和你一样、现在已走出来的人"。系统比较的是用户"现在"和候选灯塔"过去（历史节点向量）"，而非双方的"现在"。在推荐系统文献中属于"异时态语义匹配"，目前鲜有产品级实现。

### 创新三：情感安全的 AI 社交

MOLT 设计了 Agent 守门的渐进式连接机制：首次接触只能发送无内容的"信号"，双方互发后才解锁一句话交换。对于脆弱状态下的求职者，一次糟糕的陌生人互动可能造成进一步退缩。Agent 确保每一次连接都经过"资格审查"。

### 创新四：叙事驱动的 AI 流程

整个 AI pipeline 由叙事隐喻驱动：壳裂开 → 壳下面是什么 → 新壳硬化 → 碎片拼合 → 找到那个人。叙事包装直接改变了技术实现：LLM prompt 是"描述裂缝场景"而非"收集焦虑标签"，幕间动画是叙事节奏而非 loading spinner。

## 7. 核心问题

- 用户有情绪，但缺少不被审判的表达入口
- 求职焦虑状态下用户"说不出来"，传统表单/文字输入门槛高
- 用户看到大量信息，却看不到"像我这样的人后来去了哪里"
- 用户想连接真实的人，但不想进入低质量社交

## 8. 产品假设

- 如果用选卡片替代填写文字，在焦虑状态下的完成率会更高
- 如果 Agent 告诉用户"你和 47 个人正在经历类似的转变"，孤独感会下降
- 如果匹配的不是"成功的人"而是"曾经和你一样、已走出来的人"，可信度会更高
- 如果首次连接只需发送一个无内容信号，社交门槛会足够低

## 9. Hackathon 范围

**必须跑通：**
- 三张卡片路径选择
- 脱壳三幕（三次卡片点击，约65秒，零文字输入）
- 画像合成 + 语义聚类展示
- 节点匹配（三维评分：语义50% + 时序30% + 方向20%）
- 匹配结果 + 灯塔留言
- 信号发送

**加分项（已全部达成）：**
- 可运行 Demo ✅（完整 Web 应用，从记忆导入到匹配结果全流程）
- 真实用户验证 ✅（赛期内≥3名真实用户完整使用反馈）
- 技术复杂度 ✅（多模型协作 + 语义向量匹配 + 时序互补引擎）
- 开源贡献 ✅（GitHub 公开仓库，MIT协议）

- 关键链路在 AI 失败时通过降级卡片和种子数据继续演示

## 11. 技术选型摘要

- **前端：** React + TypeScript + Tailwind CSS
- **服务端：** Supabase Edge Functions（无服务器）
- **数据库：** PostgreSQL（Supabase，含 pgvector 扩展）
- **AI：** Claude Sonnet / OpenAI（.env 双兼容）
- **向量引擎：** text-embedding-3-small（1536维）
- **种子数据：** 31个灯塔节点（静态 JSON + 预计算向量）
- **会话状态：** LocalStorage

## 12. 风险与应对

| 风险 | 应对 |
|------|------|
| AI 调用链过长 | 单用户仅5-7次调用，约5,000 tokens；降级卡片池保障核心流程 |
| LLM 出题慢 | 叙事动画（"裂缝正在扩大..."）遮盖1-2秒延迟，变成叙事体验 |
| 灯塔节点池太小 | 预置31个种子节点（静态JSON + 预计算向量），确保总有匹配 |
| 脆弱用户被低质量连接伤害 | Agent 守门机制：只能发信号，不能主动发消息；出现危机信号停止匹配 |
| 路演模式 AI 不可用 | `DEMO_MODE=true` 直接返回预设结果，全流程可跑 |

## 13. 商业化方向

- **C端免费：** 脱壳三幕、匹配推荐、信号连接
- **B端付费：** 高校就业指导中心（节点匹配引擎年费订阅）、求职平台（MOLT API 集成）
- **数据壁垒：** 真实用户轨迹数据，无法爬取、无法合成，越积累越有价值
- **规模化路径：** 种子期（500用户，5所高校）→ 高校扩展 → 品类扩展（考研/留学/转行）→ 平台化（MOLT API）
