# MOLT Onboarding 交互设计文档

**产品**：MOLT（脱壳）  
**模块**：三张卡片选择式入场 + 三条路径完整交互设计  
**版本**：v1.0  
**日期**：2026-03-20  
**作者**：王文钦 Ellie  

---

## 0. 设计原则

MOLT 的 onboarding 核心命题只有一个：**让用户在进入产品的第一分钟，感到被看见。**

三张卡片选择式入场，是整个 onboarding 的第一屏。它不要求用户填写任何信息，只需要做一个选择：认出自己。

---

## 1. 第一屏：三张卡片选择

用户进入 MOLT，看到三张卡片，每张是一段话，不是一个标签。

---

### 卡片一｜挣扎（Path A）

```
「我知道自己的专业要消失了
  但我不知道下一步在哪里」

我有点像这个 →
```

---

### 卡片二｜挣脱（Path B）

```
「我已经在行动了，但不确定
  自己走的方向对不对」

我有点像这个 →
```

---

### 卡片三｜回望（Path C）

```
「我走出来了，但我想让
  后来的人少走弯路」

我有点像这个 →
```

---

### 选择逻辑

用户选择一张卡片后，系统根据选择决定：

- 后续三幕对话的**语气**（迷茫型 / 探索型 / 灯塔型）
- 置换压力指数的**权重侧重**
- Agent 匹配的**对象类型**

---

## 2. Path A｜正在换壳

**语气基调：慢、温、不催促。系统在陪你，不是在处理你。**

---

### 步骤一：情绪锚定追问

用户选了 Path A，系统不问「你的专业是什么」，而是：

**第一问（固定）：**

> 「在你开始感到不确定之前，有没有一个具体的时刻？」
>
> *「可以是一条新闻，一次对话，一个夜晚。不需要完整，说一句就够。」*

**第二问（动态生成，嵌入用户原话）：**

> 「你说了『___』——那个时候，你最想知道的是什么？」

**第三问（固定）：**

> 「如果现在有一个人，六个月前和你站在完全一样的地方，你最想问她一句什么话？」

**第三问的作用：** 提取用户真正需要的连接类型（情感支持 / 路径参考 / 技能建议），直接影响后续 Agent 匹配逻辑。

---

### 步骤二：镜子确认（Path A 专属版本）

镜子复述不做简历式总结，而是情绪式复述：

```
我听到了这些——

你说你「___」（用户原话片段）
你说你不知道「___」（提取的核心困惑）
你问的那个问题，其实是：「___」（系统重构的真实问题）

这说的是你吗？
```

- 主按钮：**「是的，继续」** → 进入结果页
- 次按钮：**「有些不对」** → 回到追问第二幕，重新生成

---

### 步骤三：置换压力区间（Path A 专属拆解方式）

结果页首屏压力区间大字出现，拆解语言比其他路径更温：

```
你的置换压力区间：70 ~ 80

其中，有多少是你的问题？

结构性压力（60分）——
「整个外语翻译方向都在收缩，这不是你一个人的问题，
  是这个行业正在经历的事。你感到的压力，是真实的。」

个体行动区间（18分）——
「但有一部分，是你还没有发现自己的隐性资产。
  你在第二问里提到了___，这件事在简历上消失了。」
```

**能力重估模块标题（Path A 专属）：**「你以为没用的，其实值多少」

---

### 步骤四：Agent 匹配

**匹配逻辑：** Path A 优先匹配 Type C（灯塔）节点——走过来的人，而非同样在路上的人。

**Agent 输出文案模板：**

> 「我们找到了一个人。___个月前，她的处境和你几乎一样：___（一句话描述相似点）。她现在在做___。她愿意和你说一句话。」

**发送信号按钮文案：**「我想听她说的那句话」

---

## 3. Path B｜挣脱

**语气基调：平等、务实、快节奏。系统在帮你校准，不是在安慰你。**

---

### 步骤一：方向校准追问

**第一问（固定）：**

> 「你在往哪个方向走？用一句话描述你现在正在尝试的事。」
>
> *「不需要说成一个成熟的计划，说正在做的动作就够了。」*

系统从回答里提取：方向标签（产品 / 运营 / 技术 / 内容 / 研究…）、当前阶段（学习 / 实习 / 求职 / 在职转型）。

**第二问（基于第一问动态生成）：**

> 「你怎么判断自己走对了还是走偏了？有没有一个你拿不准的具体信号？」

**第三问（固定，含分支）：**

> 「在这条路上，你最需要的是什么——一个数据（市场真的需要这个吗），还是一个人（有人走过这条路吗）？」

选项：**[给我数据]** / **[给我一个人]**

**分支逻辑：**
- 选「数据」→ 结果页优先展示岗位趋势与市场信号
- 选「一个人」→ 结果页优先展示 Agent 匹配

---

### 步骤二：镜子确认（Path B 专属版本）

B 的镜子更像「方向诊断摘要」：

```
你正在做的事：___
你拿不准的信号：___
你需要的验证类型：___（数据 / 人）

这说的是你现在的状态吗？
```

- 主按钮：**「是的，继续」**
- 次按钮：**「我来修正」**

---

### 步骤三：方向验证结果页（Path B 专属）

置换压力区间的拆解方式：

```
你的置换压力区间：55 ~ 65
（你已经在移动，区间比起点的人低）

你正在走的方向——

市场信号（基于岗位快照数据）：
「___方向过去 90 天岗位___% ，核心需求技能是___」

你现在的能力组合和这个方向的距离：
「你有___，还缺___，最快的补位方式是___」

本周可以验证的一件事：___
```

**新增模块：** 「走这条路的人，通常经历了这几个阶段」——用 3-4 个简化轨迹节点展示，让用户看见自己在哪里。

---

### 步骤四：Agent 匹配

**匹配逻辑：** B 优先匹配同路径上比自己超前半步的人——不是已经成功的人，是正在走同一条路且走在前面一点的人。

**Agent 输出文案模板：**

> 「有个人正在走和你几乎一样的路：___。他比你早了___个月，现在到了___这个节点。他愿意和你说一句话。」

**发送信号按钮文案：**「我们走的是同一条路」

---

## 4. Path C｜回望

**这是一个完全不同的 onboarding 逻辑——系统不是在帮你解决问题，是在邀请你成为解决方案。**

**语气基调：郑重、受邀、有仪式感。**

---

### 步骤一：轨迹录入引导

Path C 的第一屏不是追问，是一段话：

```
你走出来了。

但你走过的这条路，还有人不知道它存在。

MOLT 想把你的轨迹变成地图上的一条路径——
不是成功学故事，是一段真实的「从这里，到那里」。

愿意说说吗？
```

**三个问题，顺序固定，语气更像采访：**

**Q1：**

> 「在你开始感到不确定之前，你在哪里？」
>
> *「说说当时的专业 / 职业 / 状态，一两句就够。」*

**Q2：**

> 「转折是怎么发生的？有没有一个具体的时刻或决定？」
>
> *「不需要说成一个完整的故事，说那个关键动作就够。」*

**Q3：**

> 「现在的你，想对六个月前的自己说一句什么话？」

**Q3 的作用：** 这句话将成为用户节点上展示给其他人看的那句话。

---

### 步骤二：轨迹摘要确认（Path C 专属镜子）

系统不做情绪镜子，做「轨迹摘要卡」：

```
你的过渡轨迹——

起点：___（从 Q1 提取）
转折：___（从 Q2 提取）
现在：___（系统基于整体推断）

你想对后来的人说：「___」（来自 Q3 原话）

这是你的轨迹，准确吗？
```

- 主按钮：**「是的，这就是我的路」**
- 次按钮：**「我来补充一些细节」**

---

### 步骤三：公开意愿设置

Path C 独有的步骤，其他路径没有：

```
你的轨迹将成为地图上的一个节点。

你希望其他人看到多少？

○ 完整展示（轨迹起点、转折、现状、那句话）
○ 只展示方向和那句话
○ 只展示「曾经在这里，现在走出来了」

你可以随时修改这个设置。
```

---

### 步骤四：成为地图灯塔

Path C 没有「Agent 匹配文案」，替换成一个仪式感更强的结束页：

```
你现在是地图上的一个灯塔。

当有人和六个月前的你处于同一位置时，
MOLT 会在合适的时候，把你的轨迹展示给他们。

他们不会知道你的名字，
但他们会看见你走过的路。

你的那句话已经在等他们了——
「___」（用户 Q3 的原话）
```

**底部两个可选行动：**

- **「我也想看看现在地图上有哪些人」** → 进入地图页
- **「我身边有朋友也在这条路上」** → 分享入口

---

## 5. 三条路径差异对照

| 维度 | Path A 正在换壳 | Path B 已经移动 | Path C 已经落地 |
|---|---|---|---|
| 追问语气 | 慢、温、情绪承接 | 快、平等、务实 | 郑重、受邀、采访感 |
| 镜子内容 | 情绪镜子 | 方向诊断摘要 | 轨迹摘要卡 |
| 结果页重点 | 压力拆解 + 资产发现 | 市场验证 + 距离测量 | 无结果页，直接建档 |
| Agent 匹配对象 | 灯塔（走出来的人） | 同路先行者 | 不匹配，成为被匹配者 |
| 发送信号文案 | 我想听她说的话 | 我们走同一条路 | 无（成为灯塔） |
| 情感落点 | 被看见、被接住 | 被校准、被确认 | 被需要、有意义 |

---

## 6. 边缘 Case 处理

### 用户无法选择卡片

如果用户感觉三张卡片都不完全匹配，提供第四个入口：

> **「都有一点，但又不完全是」** → 进入 Path A 流程（默认最温的路径）

### 用户中途切换路径

如果用户在 Path A 的镜子确认页点击「有些不对」超过两次，系统提示：

> 「要不要重新选择你的起点？」→ 返回三张卡片

### AI 生成失败

- 追问失败 → 回退 Ellie 预写的对应路径模板（A / B / C 各 3 套）
- 镜子失败 → 回退对应路径的镜子模板
- Agent 匹配失败 → 展示公开轨迹样本节点，标注「这是地图上的真实轨迹」

### Demo 模式

`DEMO_MODE=true` 时，指定演示账号直接返回预设的完整结果，跳过真实 AI 调用。每条路径各准备一套预设结果。

## 7. 埋点建议

| 事件名称 | 触发时机 | 关键参数 |
|---|---|---|
| `select_onboarding_card` | 用户点击三张卡片之一 | `path_type: A/B/C` |
| `submit_path_answer` | 每幕提交答案 | `path_type`, `scene_index`, `text_length` |
| `confirm_mirror` | 用户点击「是的，继续」 | `path_type`, `attempt_count` |
| `reject_mirror` | 用户点击次按钮 | `path_type`, `scene_index` |
| `select_need_type` | Path B 第三问选择 | `need_type: data/person` |
| `set_visibility` | Path C 公开意愿设置 | `visibility_level: full/partial/minimal` |
| `become_lighthouse` | Path C 完成建档 | `path_type: C` |

## 8. 文案 Tone of Voice 对照

| | Path A | Path B | Path C |
|---|---|---|---|
| 系统说话方式 | 「我听到了……」 | 「你在做……」 | 「你走出来了……」 |
| 避免出现 | 「建议你……」 | 「你应该……」 | 「你成功了……」 |
| 核心情绪 | 被接住 | 被看清 | 被需要 |
| 结尾记忆点 | 那句被接住的话 | 那个被校准的方向 | 那句留给后来者的话 |
