# MOLT 交互原型说明

## 1. 目标

本文件用于定义 Hackathon 版本的页面流转、关键组件和交互规则。当前采用低保真文字原型，优先服务开发与演示。

## 2. 页面流转

```text
Landing
  -> 三幕对话
  -> 镜子确认
  -> 结果页（压力区间 + Agent 文案）
  -> 能力重估卡片
  -> 地图样例页
  -> 节点详情 / 信号
```

## 3. 页面原型

### 3.1 Landing

- 页面目标：
  - 用一句产品宣言完成情绪抓手
  - 让用户只做一个决定：开始
- 组件：
  - 主标题：`你不是被时代淘汰的人。你是正在换壳的人。`
  - 副标题：一段 1 至 2 行的承接说明
  - CTA：`开始脱壳`
- 交互规则：
  - 页面不出现复杂导航
  - 滚动深度控制在 1 屏到 1.5 屏

### 3.2 三幕对话

- 页面目标：
  - 让用户进入非问卷式自白状态
- 组件：
  - 当前问题卡片
  - 多行输入框
  - 提交按钮
  - 进度提示：`第 1 / 3 幕`
- 交互规则：
  - 一次只出现一个问题
  - 输入为空时不可提交
  - 提交后按钮进入 loading，并禁止重复点击

### 3.3 镜子确认

- 页面目标：
  - 让用户确认“这就是我”
- 组件：
  - 镜子文案卡片
  - 主按钮：`这就是我`
  - 次按钮：`换种说法`
- 交互规则：
  - 主按钮进入结果页
  - 次按钮返回追问页并重新生成镜子文案

### 3.4 结果页

- 页面目标：
  - 输出一眼能懂的判断和一句强记忆点文案
- 组件：
  - 压力区间主卡
  - 结构性压力说明
  - 个体行动说明
  - Agent 推荐文案卡
- 交互规则：
  - 首屏必须完整展示压力区间和 Agent 文案
  - 能力重估卡片可位于首屏下方

### 3.5 能力重估卡片

- 组件：
  - 正在贬值的能力
  - 被低估的迁移能力
  - 推荐岗位方向
  - 本周最小行动
- 交互规则：
  - 每块内容不超过 3 条，避免信息过载

### 3.6 地图样例页

- 页面目标：
  - 用可视化展示“你不是孤点”
- 组件：
  - 节点图谱区域
  - 图例说明
  - 节点详情抽屉
- 交互规则：
  - 首版使用预填充静态节点
  - 点击节点打开侧边抽屉

### 3.7 节点详情 / 信号

- 组件：
  - 节点标签
  - 简化轨迹文案
  - 按钮：`发送信号`
- 交互规则：
  - 发送成功后按钮进入已发送态
  - 不展开长聊天能力

## 4. 核心状态

- 空状态：对话未开始、地图无节点、无匹配对象
- 加载状态：AI 追问生成中、镜子生成中、结果页生成中
- 降级状态：使用模板追问、模板镜子、样例结果
- 异常状态：超时、生成失败、信号发送失败

## 5. 演示顺序建议

建议路演时只沿以下顺序操作：

1. Landing
2. 三幕对话
3. 镜子确认
4. 结果页
5. 地图样例页

避免在主流程里展示过多分支，以保证节奏和稳定性。
