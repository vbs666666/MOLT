# MOLT UI 设计说明 / Design Guide

## 1. 设计目标 (Design Goals)

MOLT 的 UI 需要在视觉感知上同时完成三件事：

- **承接情绪，但不落入“疗愈 App”的陈词滥调**：用户是来寻求分析和破局的，而非单纯的安慰。
- **体现结构感和判断力，摒弃泛 AI 聊天感**：界面体验应该像是一个精密、冷静且强大的“心理诊断引擎”，而非普通的对话框。
- **在 Hackathon 场景下形成强记忆点**：用极具张力的前沿视觉风格打破常规，一秒抓住评委眼球。

***核心平衡点**：在“专业硬核”的黑色极简骨架下，通过细腻的微弱发光(Glow)与缓动交互(Easing)提供人性化的包容感。*

## 2. 视觉关键词 (Visual Keywords)

- 克制 (Restrained)
- 锋利 (Sharp)
- 呼吸感 (Breathable)
- 夜间质感 (Cyber-Minimalism / Dark Mode)
- 结构化分析 (Structural / Node Network)

## 3. 设计令牌：配色方案 (Design Tokens: Colors)

整体采用 **高对比度的暗色线性美学 (Linear-Style)**：

- **背景层 (Background)**
  - `bg-base`：`#0A0A0A` (极深黑，沉浸式的全屏底层底色)
  - `bg-surface`：`#141414` / `rgba(255,255,255,0.03)` (卡片容器底色，微弱反光的磨砂质感)
- **文字层 (Typography)**
  - `text-primary`：`#F5F5F5` (高亮白内容，用于主体核心文案、判定结果)
  - `text-secondary`：`#A3A3A3` (次级功能说明、时间戳、解析状态)
- **强调色 (Accents - 霓虹质感)**
  - `accent-green`：`#7CFF6B` (高显色亮绿，传达：主干路、明确结果、积极建议、Agent确认)
  - `accent-blue`：`#5BE7FF` (冰裂蓝，传达：系统连接中、逻辑重构、结构化分形)
  - `accent-red`：`#FF5C5C` (风险深红，严格控制使用面积，仅用于严重压力警告)
- **描边与分割线 (Strokes)**
  - `stroke-hairline`：`rgba(255, 255, 255, 0.08)` (极细边框线，勾勒出现代极简的结构感)
  - `stroke-focus`：`rgba(124, 255, 107, 0.4)` (焦点态晕染范围，例如输入框 Hover，带来“被接纳感”)

## 4. 字体与排版体系 (Typography)

采用 **“数据系统” + “人文阅读”** 的字体混排策略：

- **英文与数字/数据标签 (Data & Code)**：
  - 推荐：`JetBrains Mono` 或 `Geist Mono` 等等宽字体。
  - 理由：强化“系统演算引擎”的硬核感与节点梳理感。
- **中文正文 (Body & Reading)**：
  - 推荐：现代无衬线黑体 (苹方 `PingFang SC`, 苹方简 或 `Inter` 对应中文字体族)。
  - 节奏：
    - 大段落引导语、宣言句：`Medium (500)` 或 `Semibold (600)`
    - 附属说明、节点标签：`Regular (400)`。严格控制字体的呼吸感与行距 (`Line-height: 1.6 - 1.8`)。

## 5. 核心页面信息层级 (Information Hierarchy)

- **H1 核心视点**：产品 Slogan 或最终生成的【结构性结论】。极大字号，极强对比。
- **H2 模块划分标题**：例如“情绪溯源”、“被低估的资产”。
- **Body 躯干内容**：系统拆解的说明和节点文字，需要长篇阅读时保证对比度和舒适度。
- **Caption 附属信息**：标签、时间、Agent 思考过程的占位符。

## 6. 组件规范 (Component System)

摒弃传统且厚重的 UI 控件，拥抱现代 Web 端点（Edge）设计语言：

### 6.1 浮态卡片 (Glass Cards)
- 背景使用**弱背景模糊 (Backdrop-filter: blur(12px) - 24px)**。
- **内边距 (Padding)**：严格遵守 `20px` 或 `24px`，保持留白。
- **圆角 (Border Radius)**：锁定 `16px`。
- **光泽细节**：卡片上边缘可加上 `1px`，`rgba(255,255,255,0.15)` 的内反光高光边缘，增强玻璃质感。

### 6.2 交互按钮 (Buttons)
- **主发光操作（如开始解构/提交分析）**：全白或全绿底色，反色文字。
- **次级操作（如重新探索）**：通透的暗色底板 + `0.08` 透明度的描边，Hover 时文字微亮。
- **禁用态 (Disabled)**：透明度直接跌落至 `40%`，无任何 Hover 动效。

### 6.3 沉浸式输入终端 (Input Terminal)
- **抛弃传统的表单堆叠设计**，只给出一个宽大空旷的输入画布。
- **状态响应**：没有框线的压迫感，当光标聚焦 (Focus) 时，底部或边框微微泛起 `accent-blue` 或 `accent-green` 的呼吸光晕。

### 6.4 节点力导向图谱体系 (Node Canvas)
- **样式**：不使用花哨渐变的节点实体，最好是实线圈出空心节点。
- **强调**：路线通过深色/浅色区分阶段；当前用户聚焦的节点亮度最高。
- **边线**：极弱透明度的灰线连结系统，不喧宾夺主。

## 7. 动效与微交互法则 (Motion & Micro-interactions)

动效绝不可堆砌，每一次动画都必须服务于“系统正在深思和重构”的隐喻：
- **核心曲线**：拒绝默认线性的僵硬动画，推荐使用定制缓动 `cubic-bezier(0.2, 0.8, 0.2, 1)` 或 Spring(弹簧) 阻尼动效。
- **必备动效 Top 3**：
  - **首屏 Landing**：大标题与控制终端以阶梯式的**从下至上微浮淡入 (Fade-in-up)**。
  - **结论生成态**：百分比数字飞速跳动或关键卡片“逐字打印 (Typewriter) + 光标闪烁”。
  - **节点漫游**：图谱节点在坐标轴上的轻微呼吸律动。

## 8. 可访问性与落地性能防坑 (Accessibility & Performance)

- **对比度底线**：即使追求极致的次级文字“暗夜感”，`#A3A3A3` 在极黑 `#0A0A0A` 上的对比度依然能够过及格线，严禁为了设计把字色降到 `0.3` 或隐形。
- **性能降级策略 (Hackathon Survival)**：节点图谱容易成为性能毒药。在赶工时或移动端，建议将复杂的全物理引擎画布**降级**为“静态背景纹理 + CSS Hover 动效高亮节点”。

## 9. 坚决避免的雷区 (Anti-Patterns)
- ❌ **明亮渐变的治愈风（糖果色/水彩风）**：极度违和，不符合破局诊断的专业调性。
- ❌ **过度拟物的聊天气泡（微信/iMessage 对话式 UI）**：这会让应用彻底平庸化，缺乏结构性审视。
- ❌ **拥挤的顶栏/侧边栏**：保持单一专注力，隐藏不必要的层级。
- ❌ **首屏直接铺大段阅读说明文本**：用留白和卡片引导用户动作。

## 10. Hackathon 交付优先级 (Hackathon Sprint)

时间紧迫，集中火力点亮 MVP：
- ⭐️ **必须拿下 (P0)**：
  - Landing 首屏的硬核极客感 (震撼评委第一眼)。
  - 输入情绪的终端交互态。
  - 结构性出具最终诊断卡与结论（大字号黑白绿搭配）。
- 🟡 **次优增强 (P1)**：
  - 各类能力维度的多维坐标图腾。
  - 画布式的“情绪演进到资产演进”二维地图背景骨架（静态图也是可行的包装）。
- 📉 **留给彩蛋 (P2)**：
  - 极细的节点拖拽、力导向引擎碰撞。
  - 页面翻转/元素魔术般切换等耗时动画。
